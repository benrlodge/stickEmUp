Stick Em Up! (.js)

Flexible, simple, sticky stuff

I created Stick Em Up so I could have a flexible and easy way to set fixed positioning on elements based on where visitors are scrolling on webpages.

Options

While setting measuring elements offset from the top of the page, and measuring when to trigger a class when it hits a certain point seems simple enough, things get a little trickier when the size of those elements is dependent on some JavaScript that needs to be loaded. For an example, you may want to have a fixed set of social sharing icons that follows you down a page. Thing is, you need the Twitters, Facebooks, LinkedIns and what nots to load their iframes in (if you go that route) before you will know the exact height of those elements. Sure, you can hardcode the expected size within the CSS of the container, but where's the fun in that??

About

Setting elements sticky is a frequently asked for feature. As a resourceful developer I looked for tools that would help me do this without writing writing my own. I was hopeful when I found the waypoints plugin. Unfortunately, I found lots of funny behavior during window resizing, so I decided to just go ahead and create my own. Waypoints has many more features, this plugin is much simpler. It will tell you when your given element is at a certain point on the page, what direction the user was scrolling when it hit that element, adds a class to the element during that time, and provides a callback throughout all of this scrolling to do what you please with this information.

This is all I've needed for my basic needs, so that's all it provides. If you have any ideas for additional features, please feel free to submit a Github issue.
